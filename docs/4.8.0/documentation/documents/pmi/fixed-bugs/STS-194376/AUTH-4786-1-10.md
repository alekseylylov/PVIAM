# Корректное имя куки для стики в CDL при пустом значении iamproxy.k8s.session_namek8s.session_name

## Описание

Была ошибка при которой spec.trafficPolicy.loadBalancer.consistentHash.httpCookie.name заполнялось неверное при пустом значении iamproxy.k8s.session_namek8s.session_name
По умолчанию k8s.session_namek8s.session_name имеет значение `PLATFORM_SESSION`. Имя сессионного `cookie`

## Предусловия

В параметрах деплоя оставить пустым **iamproxy.k8s.session_namek8s.session_name**

## Шаги проверки

1. **Действие**:

   Перейти на страницу прокси и открыть DevTools (F12)

   **Успешный результат**:

   Выполнен редирект на страницу аутентификации

2. **Действие**:

   Пройти аутентифифкацию

   **Успешный результат**:

   Аутентификация успешно пройдена

3. **Действие**:

   В DevTools перейти на вкладку application и открыть cookie

   **Успешный результат**:

   В разделе cookie нет куки с именем "_ID"
