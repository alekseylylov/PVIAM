# Удаление параметров ответвления (junction)

## Проблематика  
Администраторы сталкиваются с необходимостью удаления устаревших или ненужных ответвлений (junction) из конфигурации IAM Proxy. Некорректное удаление может привести к нарушению работы связанных сервисов или остаточным конфигурациям, влияющим на производительность.

## Предусловия  
1. **Доступ к среде**:  
   - доступ к интерфейсу PACMAN (CFGA) с правами администратора;  
   - знание расположения и структуры YAML-файлов конфигурации IAM Proxy.  

2. **Требуемые навыки**:  
   - опыт работы с PACMAN (CFGA) и редактированием конфигурационных файлов;  
   - понимание параметров настройки `JunctionConfig` и их влияния на работу IAM Proxy (см. [Параметры настройки](config-parameters.md)).  

3. **Инструменты**:  
   - текстовый редактор для редактирования YAML-файлов;  
   - утилиты для проверки синтаксиса YAML (например, `yamllint`).  

## Правила эксплуатации  
1. **Безопасность конфигураций**:  
   - все изменения в конфигурациях должны проходить через PACMAN (CFGA) или версионный контроль (Git);  
   - используйте TLS для всех API-запросов к IAM Proxy.  

2. **Форматы данных**:  
   - конфигурационные файлы должны соответствовать схеме YAML;  
   - при использовании PACMAN (CFGA):  
     ```yaml
     junctions:
       - name: "new-junction"
         zone: "ZoneA"
         config:
           endpoint: "https://api.new-service.com"
           timeout: 30s
     ```

3. **Особенности настройки**:  
   - при удалении ответвления в нескольких контурах (Zone) убедитесь, что удаление произведено во всех соответствующих секциях;  
   - перед удалением проверьте зависимости ответвления в других компонентах (например, в мониторинге MONA).  

## Последовательность выполнения  

### Способ 1: Через PACMAN (CFGA)  
1. **Открыть UI PACMAN (CFGA)**:  
   - выберите артефакт `rds-server` для текущего контура.  

2. **Удалить ответвление**:  
   - найдите нужное ответвление в списке `JunctionConfig`;  
   - нажмите кнопку "Удалить";  
   - если ответвление существует в нескольких контурах (Zone), повторите удаление для каждого контура.  

3. **Сохранить изменения**:  
   - нажмите "Сохранить" и дождитесь применения настроек.  

### Способ 2: Через YAML-файл  
1. **Редактировать конфигурационный файл**:  
   - откройте YAML-файл, связанный с IAM Proxy;  
   - найдите секцию `JunctionConfig`, содержащую удаляемое ответвление;  
   - удалите строку или блок с этим ответвлением. Пример:  
     ```yaml
     junctions:
       - name: "old-junction"  # <-- Удалить эту строку
         zone: "ZoneB"
         config:
           endpoint: "https://api.old-service.com"
     ```  

2. **Применить изменения**:  
   - сохраните файл и перезагрузите конфигурацию IAM Proxy (через команду `reload` или автоматическую синхронизацию PACMAN).  

## Результат  
Ответвление (junction) удалено из конфигурации IAM Proxy. Проверьте:  
- отсутствие ссылок на удаленное ответвление в логах и метриках MONA;  
- корректное поведение оставшихся ответвлений;  
- отсутствие ошибок в конфигурационных файлах после удаления.  

## Описание механизмов безопасности  
1. **Ограничение доступа**:  
   - доступ к удалению конфигураций ограничен правами администратора; 
   - все API-запросы к IAM Proxy защищены TLS 1.2+/mTLS.  

2. **Целостность данных**:  
   - YAML-файлы хранятся в зашифрованном виде в SecMan;  
   - пример типовой конфигурации после удаления:  
     ```yaml
     junctions:
       - name: "secure-junction"
         zone: "SecureZone"
         config:
           endpoint: "https://api.secure-service.com"
           auth:
             type: "mTLS"
             cert_path: "/etc/secrets/client.crt"
             key_path: "/etc/secrets/client.key"
     ```

3. **Логирование и аудит**:  
   - все изменения в конфигурациях фиксируются в журнале PACMAN;  
   - активируйте мониторинг изменений через MONA для отслеживания активности.
