# Прочие аспекты

## Сценарии отказа

В таблице отражены сценарии отказа компонентов и/или внешних элементов, от которых зависит программный продукт.

* __Элемент отказа__ - компонент программного компонента или внешней системы, или внешнего компонента платформы, от
  которой зависит программный компонент.
* __Последствия отказа__ - потеря функциональности программного компонента при недоступности компонента.
* __Компенсирующие механизмы__ - методы предотвращения последствий отказа при отказе компонента.

| Элемент отказа                                             | Последствия отказа                                                                  | Компенсирующие механизмы                                                                                   |
|------------------------------------------------------------|-------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------|
| RDS Client                                                 | Невозможность обновления конфигурации                                               | Развертывание нескольких экземпляров. Георезервирование.                                                   |
| IAM Proxy                                                  | Невозможность выполнения основной функции - проксирование траффика                  | Развертывание нескольких экземпляров. Георезервирование.                                                   |
| Fluent Bit                                                 | Невозможность отправки событий аудита и журналов                                    | Увеличение объема по CPU и/или HDD в зависимости от причин отказа.                                         |
| Keycloak.SE (KCSE)                                         | Невозможность взаимодействия ввиду отсутствия UI                                    | Развертывание нескольких экземпляров. Георезервирование.                                                   |
| Авторизация (AZGT)                                         | Невозможность администрирования ролей и доступов                                    | Развертывание нескольких экземпляров. Георезервирование.                                                   |
| Аудит (AUDT)                                               | Невозможность использовать внешнюю систему аудита                                   | -                                                                                                          |
| Единый коллектор телеметрии (COTE)                         | Невозможность использовать внешнюю систему аудита                                   | Отключить интеграцию с COTE. Использовать файлы журнала (на VM) или журналы среды контейнеризации (на K8S) |
| Журналирование (LOGA)                                      | Невозможность использовать журналирование                                           | -                                                                                                          |
| Объединенный мониторинг Unimon (MONA)                      | Невозможность использовать мониторинг                                               | -                                                                                                          |
| SecMan                                                     | Невозможность использования целевых функций IAM Proxy                               | -                                                                                                          |
| Граничный прокси (IGEG)                                    | Невозможность использования граничного прокси в среде контейнеризации               | -                                                                                                          |
| Сервисный прокси (SVPX)                                    | Невозможность использования сервисного прокси в среде контейнеризации               | -                                                                                                          |
| Управление политиками (POLM)                               | Невозможность управлять функциональностью SSM и/или невозможность использования SSM | -                                                                                                          |
| Сервис ограничения (квотирования) входящих запросов (SRLS) | Невозможность применения лимитов на входящие запросы                                | -                                                                                                          |
