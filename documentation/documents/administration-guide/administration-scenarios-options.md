# Варианты администрирования

Администрирование IAM Proxy осуществляется через изменение конфигурации следующими способами:

- администрирование с использованием yaml-файла параметров `inventory-ansible`, при установке с помощью `Ansible`.
  Процесс установки описан в
  разделе [Заполнение раздела с описанием параметров ответвлений (junctions)](../installation-guide/installation.md);
- администрирование с использованием файлов параметров `auth.*.conf` и `custom_property.conf.yml`, при установке с
  помощью `DOT CDJE`. Процесс установки описан в
  разделе [Установка IAM Proxy в OpenShift/k8s при помощи инструментов Platform V DevOps Tools](../installation-guide/proxy-deploy-ose-cd.md);
- администрирование с использованием компонента PACMAN (CFGA) (данный способ является устаревшим). Компонент PACMAN (
  CFGA) используется для изменения конфигурации без повторного развертывания, и для изменения ее в real-time. Процесс
  описан в разделе [Администрирование с помощью компонента PACMAN (CFGA)](administration-pacman.md);
- администрирование с использованием yaml-файла конфигурации формата компонента PACMAN (CFGA). Используется для
  изменения конфигурации без повторного развертывания, и для изменения ее в real-time. Подробнее в
  разделе [Настройка получения конфигурации](../additional-materials/rds-server-get-conf.md);
- администрирование с использованием API RDS, который реализуется на приложении смежной системы (как и UI и средства по
  управлению API). Используется для изменения конфигурации без повторного развертывания, и для изменения ее в real-time.
  Подробнее в
  разделе [Описание API для реализации динамической настройки IAM Proxy](../additional-materials/rds-api.md).

> Примечание
> 
>Для администрирования с использованием PACMAN (CFGA) или yaml-файла формата PACMAN, требуется установка компонента
> RDS Server на стенде.
