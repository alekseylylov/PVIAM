# Аутентификация через IAM Proxy без подключения к IDP по OIDC

## Проблематика  
Приложения могут быть сложными в интеграции с OIDC-провайдерами (например, Keycloak, Auth0) из-за ограничений в архитектуре, лицензий или отсутствия ресурсов для настройки. При этом требуется обеспечить аутентификацию пользователей без модификации самого приложения.

## Предусловия  
1. IAM Proxy уже настроен как центральная точка аутентификации.  
2. Приложение размещено за IAM Proxy и использует его для всех входящих запросов.  
3. IAM Proxy может добавлять HTTP-заголовки с информацией о пользователе (например, `X-User-ID`, `X-Auth-Token`).  

## Последовательность выполнения  

1. **Настройка IAM Proxy**  
   - в конфигурации IAM Proxy включите автоматическое добавление заголовков с данными аутентифицированного пользователя. Например:  
     ```yaml
     proxy:
       headers:
         add:
           X-User-ID: "{{ .Claims.sub }}"
           X-User-Name: "{{ .Claims.preferred_username }}"
     ```

2. **Изменение логики приложения**  
   - приложение должно перестать использовать OIDC-токен и вместо этого извлекать данные из заголовков, добавленных IAM Proxy;  
   - пример проверки пользователя в приложении:  
     ```python
     user_id = request.headers.get('X-User-ID')
     if not user_id:
         return "Unauthorized", 401
     ```

3. **Тестирование**  
   - выполните запрос через IAM Proxy к приложению. Убедитесь, что заголовки с данными пользователя корректно передаются;  
   - проверьте, что приложение отклоняет запросы без заголовков `X-User-ID`.  

## Результат  
Приложение получает данные аутентифицированного пользователя из заголовков, добавленных IAM Proxy, без необходимости интеграции с OIDC-провайдером. Это снижает сложность разработки и позволяет использовать IAM Proxy как централизованную точку аутентификации.
