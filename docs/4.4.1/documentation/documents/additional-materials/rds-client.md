# RDS Client

RDS Client - клиент для получения данных от RDS Server о текущей конфигурации, и формировании файлов по шаблонам.

## Описание файлов RDS Client

### Файлы в основном в каталоге (где размещен JAR)

`rds-client-conf.properties` - конфигурационный файл.

Содержит параметры:
- `rds.read_data_from_rds_time_in_sec` - частота опроса сервиса RDS (в секундах). Не обязательный параметр, по
  умолчанию - 5 секунд;
- `rds.urls` - адреса сервиса RDS (может быть несколько, перечислять через ';'). Адрес, указанный первым, будет
  считаться основным. Пример: "https://mycompany-auth-svc-idp1-dev2.ca.mycompany.ru:7443/rds-for-proxy/active-conf-json;
  http://localhost:8080/rds-for-proxy/active-conf-json".
  Можно указать как свойство при запуске JVM, этот вариант будет иметь приоритет. При отсутствии этого свойства
  или пустом его значении процесс RDS Client завершит свою работу, после того как загрузит и обработает стартовую
  конфигурацию;
- `rds.backup_working_limit_time_in_sec` - время работы запасных адресов (в случае отказа основного), в секундах. Не
  обязательный параметр, по умолчанию - 60 секунд;
- `rds.default_dir_values` - папка по умолчанию, в которой будет выполнен поиск файлов для загрузки значения
  параметра, содержащего спецсимвол `@`;
- `templates.source_dir` - директория хранения файлов - шаблонов. (Например: "D:\\some\\dir" или "/some/dir");
- `templates.destination_dir` - директория, куда будут сохраняться файлы, сформированные на основе шаблонов.
  Например: "D:\\some\\dir" или "/some/dir");
- `templates.prefix_make_once` - префикс для шаблонов, на основе которых будут сформированы файлы лишь единожды,
  при запуске. Например префикс "template". Тогда, файлы шаблонов именовать как "template.*";
- `templates.prefix_make_every_time` - префикс для шаблонов, на основе которых будут при каждом изменении полученной
  конфигурации. Например префикс "jct". Тогда, файлы шаблонов именовать как "jct.*";
- `log.dir` - директория для хранения логов. Например: "D:\\some\\dir" или "/some/dir";
- `log.level` - уровень логирования. Не обязательный параметр, по умолчанию - info;
- `app.cache_dir` - директория для хранения временных файлов/файлов для отладки. Например: "D:\\some\\dir" или
  "/some/dir". Можно указать как свойство при запуске JVM, этот вариант будет иметь приоритет;
- `app.path_to_start_conf_json` - файл из которого загрузить конфигурацию при старте RDS Client. Например:
  "D:\\some\\dir\\start-conf.json" или "/some/dir/start-conf.json";
- `app.debug_mode` - режим работы приложения. При включенном режиме debug будут формироваться дополнительный файлы
  для отладки. Не обязательный параметр, по умолчанию - false;
- `param.*` - параметры для формирования файлов на основе шаблонов. Префикс параметров должен быть "param". Само
  имя параметра не должно содержать специальные символы, так как шаблонизатор может принять их за операторы
  рекомендуется разделение слов через '_' или camelCase и/или подобных, например: param.some_params.

### Папка "templates" содержит файлы-шаблоны их шаблоны именований

При обработке шаблонов используется шаблонизатор jinja2. Шаблоны должны именоваться следующим образом
`prefix.name-template.*`, где:

- `prefix` - префикс файла-шаблона. Может быть пустым, если в настройках префикс не задан, например:
  template.name-template.* или name-template.*);
- `name-template` - имя файла с шаблоном-именованием. Данный файл создает имя для файлов по шаблону. Если шаблона для
  именований нет - то файл будет с тем же именем, что и его шаблон.

Пример каталога с шаблонами (при префиксе "template"):

- `template.upstream-name.upstream.j2` - файл-шаблон upstream. Использует для именования файл upstream-name;
- `upstream-name` - шаблон-именование;
- `template.server-name.server.j2` - файл-шаблон server. Использует для именования файл server-name;
- `template.server-name.another-server.j2` - файл-шаблон another-server. Использует для именования файл server-name;
- `template.upstream` - файл-шаблон upstream. Не имеет файла именования, поэтому в результате получим одноименный файл
  upstream;
- `server-name` - шаблон-именование.

### Особенности

Если содержимое шаблона начинается с последовательности `#jinja2:`, то вся первая строка просто удаляется из шаблона.

При обнаружении спецсимвола `@` в значении параметра будет выполнена попытка загрузки значения из файла:

- если в значении параметра указан один спецсимвол `@`, то выполняется попытка чтения из файла с именем совпадающим со
  значением имени параметра;
    - пример: `key=@` => значение будет вычитано из файла `./key`;
    - чтобы назначить папку по умолчанию, в которой будет выполнен поиск файлов, необходимо задать в файле конфигурации
      значение для параметра `rds.default_dir_values`;
    - если не задать параметр `rds.default_dir_values`, то папкой по умолчанию станет та, в которой находится файл
      конфигурации RDS-клиента.
- если будут указаны символы после спецсимвола `@`, то они принимаются за путь до файла.
    - пример: `key=@/some/dir/file` => значение будет вычитано из файла `/some/dir/file`.

## Запуск

```
# база доверенных сертификатов, открытые ключи (ЦС и/или серверов)
JAVAOPTS="$JAVAOPTS -Djavax.net.ssl.trustStore=trust-store.keystore"
JAVAOPTS="$JAVAOPTS -Djavax.net.ssl.trustStorePassword=password"
# база личных сертификатов, закрытые ключи
JAVAOPTS="$JAVAOPTS -Djavax.net.ssl.keyStore=key-store.p12"
JAVAOPTS="$JAVAOPTS -Djavax.net.ssl.keyStorePassword=password"
# задаем псевдонима сертификата в базе (javax.net.ssl.keyStore) для его принудительного использования
# иначе сертификат определяется по ЦС
JAVAOPTS="$JAVAOPTS -Djavax.net.ssl.keyAlias=mycompany-auth-svc-proxy-dev2.ca.mycompany.ru"
# разрешить использование самоподписанных сертификатов как доверенных
JAVAOPTS="$JAVAOPTS -Djavax.net.ssl.allowSelfSigned=true"
# задать стартовую конфигурацию из файла
JAVAOPTS="$JAVAOPTS -Dapp.path_to_start_conf_json=/temp/rds-client/start-conf.json"
# отключить периодическое обращение к RDS Server, и выход после обработки стартовых операций
JAVAOPTS="$JAVAOPTS -Drds.urls="

# включение отладки SSL/mTLS
#JAVAOPTS="$JAVAOPTS -Djavax.net.debug=ssl"

java $JAVAOPTS -jar "rds-client.jar" "/tmp/rds-client/my-rds-client-conf.properties"
# или запуск с файлом property по умолчанию "rds-client-conf.properties" из каталога где находится rds-client.jar
java $JAVAOPTS -jar "rds-client.jar"

```

### Коды завершения процесса

- `0` Завершение без ошибок.
- `1`..`200` Коды ошибок обновления конфигурации в SynGX.
- `201` Ошибка в главном потоке.
- `202` Прерывание выполнения скрипта.
- `203` Ошибка ввода-вывода при выполнении скрипта.
- `204` Ошибка создания шаблона скрипта.
- `211` Ошибка создания шаблона с новым контекстом при обработке массива значений.
- `212` Ошибка создания шаблона при обработке одиночного шаблона.
- `213` Ошибка доступа к шаблону для формирования имени целевого файла.
