# Token_Reuse_Pattern

## Контекст
В системах, где IAM Proxy используется как центральная точка аутентификации, часто возникает необходимость передавать полученные токены дальше по цепочке вызовов между сервисами.

## Проблема
Если каждый сервис должен повторно проверять токен, это приводит к дублированию усилий и потенциально к снижению производительности.

## Решение
Реализовать паттерн "Token Reuse", позволяющий IAM Proxy добавлять оригинальный токен в заголовок ответа (`X-Original-Token`) или передавать его дальше в виде `Authorization` заголовка при вызовах внутренних сервисов. Это позволяет внутренним сервисам использовать этот токен для дополнительной проверки прав, если необходимо, без необходимости повторной аутентификации.
