# Authentication_Bypass_Pattern

## Контекст
IAM Proxy используется в качестве шлюза аутентификации и авторизации для микросервисной архитектуры. В некоторых сценариях требуется обеспечить прозрачный доступ к сервисам без участия конечного пользователя, например, при взаимодействии между сервисами или при выполнении системных задач.

## Проблема
В таких случаях использование стандартной формы аутентификации (например, через OAuth2) может быть избыточным, увеличивая нагрузку на систему и усложняя логику обработки запросов.

## Решение
Реализовать паттерн "Authentication Bypass", который позволяет IAM Proxy определять, требуется ли выполнять аутентификацию, основываясь на заголовках запроса, пути (endpoint), источнике или типе клиента. Например:

- установить специальный заголовок `X-Internal-Request: true`, который позволит IAM Proxy пропустить этап аутентификации;
- использовать IP-листы доверенных клиентов или внутренних сервисов;
- настроить правила маршрутизации в IAM Proxy, чтобы определенные endpoints не требовали токенов.
