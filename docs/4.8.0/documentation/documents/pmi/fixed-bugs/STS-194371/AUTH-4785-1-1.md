# Использование в k8s отдельного контейнера для формирования файлов конфигурации IAM Proxy. Sidecar

## Предусловия

В параметрах деплоя auth.k8s.hotReload.enabled=True

## Шаги проверки

1. **Действие**:

   Выполнить установку IAMProxy в среде контейнеризации. Запустить POD auth-iamproxy.

   **Успешный результат**:

   Под успешно стартует

2. **Действие**:

   Перейти на вкладку "ConfigMaps" и найти конфиг auth-cm-iamproxy-opts

   **Успешный результат**:

   auth-cm-iamproxy-opts присутствует

3. **Действие**:

   Перейти в auth-cm-iamproxy-opts и в YAML добавить параметр DEBUG = '1'

   **Тестовые данные**:

   DEBUG = '1',

   **Успешный результат**:

   Параметр добавлен, под сам по себе не рестарует

4. **Действие**:

   Перейти на вкладку с подами и удалить под iamproxy

   **Успешный результат**:

   После удаления под iamproxy рестартует с новыми параметрами

5. **Действие**:

   Перейти на раздел Deployment, выбрать под с iamproxy и перейти на вкладку Environment

   **Успешный результат**:

   Успешный переход на вкладку Environment

6. **Действие**:

   В дропдауне выбрать iamproxy-conf-sidecar

   **Успешный результат**:

   iamproxy-conf-sidecar присутствует в спике

7. **Действие**:

   Добавить или изменить какой-либо параметр, присутствующий в конфигурации, например расширенный DEBUG и сохранить настройки.

   **Тестовые данные**:

   DEBUG = 2

   **Успешный результат**:

   Настройки успешно сохранены, под успешно рестартует с новыми настройками. Параметры, указанные в environment приоритетнее тех, которые указаны в ConfigMaps
