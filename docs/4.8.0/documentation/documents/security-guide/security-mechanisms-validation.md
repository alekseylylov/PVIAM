# Чек-лист валидации настройки механизмов безопасности

Раздел описывает шаги проверки механизмов безопасности реализованных в IAM Proxy.

## Чек-лист валидации настройки механизмов безопасности для VM

1. Взаимодействие с IAM Proxy осуществляется по протоколу TLS 1.2 (подробнее в разделах
   [Сетевая безопасность](network-security.md) и
   [Управление ключами и сертификатами](key-and-certificate-management.md).
2. Сертификаты хранятся в SecMan (подробнее в разделе
   [Интеграция с Vault на виртуальную машину](../installation-guide/vault-vm.md)).
3. Подключена отправка событий IAM Proxy в Audit (AUDT) (подробнее в разделе [Аудит](audit.md)).
4. Конфигурационные параметры вынесены в безопасные хранилища и хранятся в зашифрованном виде
   (подробнее в разделах [Интеграция с Vault на виртуальную машину](../installation-guide/vault-vm.md)
   и [Руководство по установке](../about/system-requirements.md)).
5. В "Junctions", опция "https" установлена в "true" (подробнее в
   разделе [Заполнение раздела с описанием параметров ответвлений](../installation-guide/installation.md)).
6. Выполнено подключение к мониторингу и в мониторинг подключена выгрузка метрик в формате `Prometheus`
   (подробнее в разделе [События мониторинга](../administration-guide/monitoring.md)).
7. В конфигурации IAM Proxy для ПРОМ-среды, значение параметра типа стенда `stend_type` установлено строго `prom` (
   подробнее в разделе [Заполнение файла в Руководства по установке](../installation-guide/installation.md)).
8. При установке IAM Proxy переменная окружения `keycloak_alternative_redirect_uri` имеет отличное от `https://*`
   значение (подробнее в
   разделе [Заполнение файла в Руководства по установке](../installation-guide/installation.md)).
9. При установке IAM Proxy, для параметра `iamproxy.k8s.oidc.introspection_interval` установлено значение `60`
   (подробнее описано в разделе
   [Соответствие имен параметров для разных сред и инструментов установки](../installation-guide/params.md)).

## Чек-лист валидации настройки механизмов безопасности для OSE

1. Взаимодействие с IAM Proxy осуществляется по протоколу TLS 1.2 (подробнее в разделах
   [Сетевая безопасность](network-security.md)
   и [Управление ключами и сертификатами](key-and-certificate-management.md)).
2. Сертификаты хранятся в SecMan (подробнее в разделе
   [Интеграция с Vault в OpenShift](../installation-guide/vault-ose.md)).
3. Подключена отправка событий IAM Proxy в Audit (AUDT) (подробнее в разделе [Аудит](audit.md)).
4. Конфигурационные параметры вынесены в безопасные хранилища и хранятся в зашифрованном виде (подробнее в разделах
   [Интеграция с Vault в OpenShift](../installation-guide/vault-ose.md)
   и [Руководство по установке](../about/system-requirements.md)).
5. В "Junctions", опция "https" установлена в "true" (подробнее в
   разделах [Заполнение раздела с описанием параметров ответвлений](../installation-guide/installation.md) и
   [Описание настройки IAM Proxy при запуске в контейнере](../installation-guide/proxy-deploy-docker-description.md)).
6. Выполнено подключение к мониторингу и в мониторинг подключена выгрузка метрик в формате Prometheus (подробнее в
   разделе [События мониторинга](../administration-guide/monitoring.md)).
7. Выполнено подключение к Platform V Synapse Service Mesh средствами администратора кластера (подробнее в разделе
   [Настройка конфигурационных параметров](../installation-guide/proxy-deploy-ose-cd.md)).
8. В конфигурации IAM Proxy для ПРОМ среды, значение параметра типа стенда `STEND_TYPE` установлено строго `prom` (
   подробнее в таблице [описания параметров в Руководства по установке](../installation-guide/params.md)).
9. В конфигурации IAM Proxy для ПРОМ среды, для параметра `iamproxy.k8s.oidc.introspection_interval` установлено
   значение `60` (подробнее описано в разделе
   [Соответствие имен параметров для разных сред и инструментов установки](../installation-guide/params.md)).

## Проверка механизмов безопасности продукта

Проверка механизмов безопасности продукта описана в документе *Руководство по установке* разделе *Чек-лист валидации установки*. 

## Проверка механизмов безопасности среды функционирования

1. **Использование безопасного протокола TLS 1.2**:  
   - подтверждение, что взаимодействие с IAM Proxy осуществляется по протоколу TLS 1.2 (подробнее в разделе [Сетевая безопасность](network-security.md));  
   - проверка корректности настройки сертификатов (подробнее в разделе [Управление ключами и сертификатами](key-and-certificate-management.md)).  

2. **Хранение сертификатов и секретов**:  
   - сертификаты должны храниться в SecMan (подробнее в разделе [Интеграция с Vault](../installation-guide/vault-vm.md) или [Интеграция с Vault в OpenShift](../installation-guide/vault-ose.md));
   - конфигурационные параметры вынесены в зашифрованные хранилища (AES-256).  

3. **Настройка аудита**:  
   - подключение отправки событий IAM Proxy в Platform V Audit SE (AUDT) (подробнее в разделе [Аудит](audit.md));
   - проверка корректности метамодели событий и их формата.  

4. **Конфигурационная безопасность**:  
   - параметр `stend_type` установлен в `prom` для промышленной среды (подробнее в [Руководстве по установке](../installation-guide/installation.md));  
   - значение `iamproxy.k8s.oidc.introspection_interval` равно `60` секунд (подробнее в [Соответствие имен параметров](../installation-guide/params.md)).  

5. **HTTPS в конфигурации ответвлений**:  
   - в разделе `Junctions` опция `https` установлена в `true` (подробнее в [Заполнение параметров ответвлений](../installation-guide/installation.md)).  

6. **Интеграция с мониторингом**:  
   - подключение метрик Prometheus для сбора данных о производительности и состоянии IAM Proxy (подробнее в [События мониторинга](../administration-guide/monitoring.md)).  

7. **Ограничение доступа к конфиденциальным данным**:  
   - проверка, что переменная `keycloak_alternative_redirect_uri` не содержит значения `https://*` (подробнее в [Руководстве по установке](../installation-guide/installation.md)).  

## Проверка механизмов безопасности во внешних сервисах

1. **KeyCloak.SE**:  
   - интеграция настроена согласно [Руководству по установке](../installation-guide/installation.md);  
   - проверка корректности передачи токенов и обновления сессий.  

2. **Авторизация (AZGT)**:  
   - настройка ролевой модели и проверка прав доступа (подробнее в [Руководстве по установке](../installation-guide/installation.md)).  

3. **SecMan**:  
   - проверка доступности секретов и корректности их использования в IAM Proxy (подробнее в [Интеграция с Vault](../installation-guide/vault-vm.md)).  

4. **mTLS**:  
   - проверка наличия действительных сертификатов в доверенных хранилищах.  

5. **Platform V Audit SE**:  
   - проверка корректности передачи событий аудита и их отображения в интерфейсе (подробнее в [Аудит](../security-guide/audit.md));  
   - проверка соответствия метамодели и полноты записей.  

6. **Synapse Service Mesh (в OSE)**:  
   - подключение к SSM выполнено средствами администратора кластера (подробнее в [Настройка конфигурационных параметров](../installation-guide/proxy-deploy-ose-cd.md));  
   - проверка межсервисной авторизации через Sidecar-прокси.
