# Удаление ответвления

## Проблематика  
Администраторам необходимо удалять устаревшие или неиспользуемые ответвления (junction) из конфигурации IAM Proxy. Некорректное удаление может привести к ошибкам маршрутизации, сбоям в работе зависимых систем, а также оставить "мертвые" конфигурации, влияющие на безопасность и производительность.

## Предусловия  
1. **Доступ к среде**:  
   - наличие прав администратора в системе управления конфигурациями или на сервере с IAM Proxy;  
   - доступ к файловой системе или UI для редактирования конфигурации IAM Proxy.  

2. **Требуемые навыки**:  
   - понимание структуры `proxy_jct_list` и соответствия полей JSON-схеме `rds-api-configuration.schema.json`;  
   - опыт работы с YAML и инструментами валидации (например, `yamllint`, `jsonschema`).  

3. **Инструменты**:  
   - текстовый редактор (VS Code, Vim и др.);  
   - утилиты проверки YAML/JSON;  
   - доступ к терминалу или API IAM Proxy для применения изменений.

## Правила эксплуатации  
1. **Безопасность конфигураций**:  
   - все изменения должны проходить через утвержденный процесс (GitOps, CI/CD или защищенный UI);  
   - запрещено удалять junction напрямую в production без предварительного тестирования в staging.  

2. **Форматы данных**:  
   - при ручном редактировании YAML-файлов убедитесь в корректности синтаксиса и отступов;  
   - после удаления не должно остаться пустых блоков или "висячих" ссылок.  

3. **Особенности настройки**:  
   - перед удалением проверьте зависимости: используется ли ответвление в мониторинге, логировании, RDS API или других системах;  
   - если junction задан в нескольких зонах или профилях — удалите его из всех;  
   - после удаления проверьте метрики и логи на предмет ошибок 404 или исключений.

## Последовательность выполнения

### Способ 1: Через внешнюю систему управления конфигурациями
1. **Открыть UI системы управления конфигурациями**:  
   - выберите артефакт, соответствующий профилю развертывания IAM Proxy для нужного стенда.

2. **Удалить ответвление**:  
   - найдите секцию `proxy_jct_list` (или аналогичную, содержащую список junction);  
   - найдите элемент с нужным `junctionPoint` и `junctionName`;  
   - удалите весь объект из массива `junctions`.

     Пример:
     ```yaml
     proxy_jct_list:
       - junctionName: "Устаревший сервис"
         junctionPoint: /old-service
         serverAddresses:
           - "old-svc.mycompany.ru:8443"
         https: true
         # <-- Этот блок подлежит удалению
     ```

3. **Сохранить и применить изменения**:  
   - подтвердите и сохраните конфигурацию;  
   - дождитесь синхронизации с IAM Proxy или инициируйте принудительную перезагрузку конфигурации.

### Способ 2: Через YAML-файл (вручную)
1. **Редактировать конфигурационный файл**:  
   - откройте файл `ansible/inventories/<StandX>/group_vars/all/main.yml`;  
   - найдите элемент в списке `proxy_jct_list` по значению `junctionPoint`;  
   - удалите весь блок ответвления, соблюдая отступы YAML.

2. **Применить изменения**:  
   - сохраните файл;  
   - перезагрузите конфигурацию IAM Proxy:

     ```shell
     curl -X POST https://iam-proxy.mycompany.ru/actuator/reload -k
     ```

## Результат  
Ответвление успешно удалено из конфигурации IAM Proxy. Проверьте:  
- отсутствие ошибок в логах IAM Proxy (`/opt/iamproxy/logs/error.log`);  
- недоступность URL ответвления (ожидается `404 Not Found`);  
- работоспособность других junction;  
- отсутствие упоминаний удаленного `junctionPoint` в системах мониторинга (MONA, Grafana).

## Описание механизмов безопасности  
1. **Ограничение доступа**:  
   - редактирование и удаление конфигураций разрешено только пользователям с правами администратора;  
   - доступ к UI и API защищен TLS 1.2+ и аутентификацией по OIDC.

2. **Целостность данных**:  
   - конфигурационные файлы хранятся в защищенных хранилищах (Vault, SecMan);  
   - перед применением выполняется валидация по схеме `rds-api-configuration.schema.json`.

3. **Логирование и аудит**:  
   - все действия по удалению фиксируются в централизованном журнале (Platform V Audit SE);  
   - рекомендуется настроить оповещения при удалении критических junction через MONA.
