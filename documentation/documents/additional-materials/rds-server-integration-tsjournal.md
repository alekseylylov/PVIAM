# Настройка интеграции с компонентом Журналирование (LOGA)

## Настройка на VM

Настройка производится через параметры профиля установки (`inventory ansible`). Профиль установки используется на этапе
развертывания компонента AUTH (установка с помощью ansible), при этом настраивается RDS Server для формирования
необходимых файлов журналов и передачу содержимого этих файлов в сервис Журналирования (передача с помощью fluent-bit).

Необходимые для интеграции параметры (с именами `logger_options.*`) и их описание смотрите в
разделе [Соответствие имен параметров для разных сред и инструментов установки](../installation-guide/params.md).

### Настройка вручную

Если требуется запустить вручную отправку в Журналирование, то для формирования логов в json-формате запускаем RDS
Server с параметром `-Dlogging.config=logback_to_files.xml` (файл расположен
в `distr/package/bh/rdsserver/Docker/root/opt/rds-server/logback_to_files.xml`). И настраиваем fluent-bit на отправку
событий из файлов `json_application_*.log` в API Журналирования, используя как пример конфигурацию из дистрибутива (
расположена в `distr/package/fluent-bit/config`).

## В OpenShift

Настройка производится через параметры профиля установки DOT (файлы `auth.*.conf`). Профиль установки используется на
этапе развертывания компонента AUTH (установка с помощью инструментов DOT), при этом в контейнере настраивается RDS
Server для формирования необходимых файлов журналов, и передача содержимого этих файлов в сервис Журналирования (
передача с помощью sidecar fluent-bit).

Необходимые для интеграции параметры (с именами `auth.k8s.logger.*`) и их описание смотрите в
разделе [Установка IAM Proxy в OpenShift/k8s при помощи инструментов Platform V DevOps Tools](../installation-guide/proxy-deploy-ose-cd).
