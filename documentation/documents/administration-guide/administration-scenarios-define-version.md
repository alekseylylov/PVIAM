# Как определить версию продукта (junction)

## Последовательность выполнения

1. **Проверьте файл конфигурации PACMAN (CFGA)**  
   - для junction-компонентов, управляемых через PACMAN: откройте артефакт `RDS Server` в PACMAN и перейдите в раздел `Конфигуратор`;  
   - используйте параметр `version` в конфигурационном файле, например:  
     ```yaml
     junction:
       version: 2.4.1
       build_date: 2023-09-15
     ```

2. **Используйте API/endpoint junction**  
   - для junction, поддерживающих REST API: отправьте GET-запрос на `/api/junction/version`;  
   - пример JSON-ответа:  
     ```json
     {
       "junction_version": "2.4.1",
       "commit_hash": "a1b2c3d4",
       "build_time": "2023-09-15T10:00:00Z"
     }
     ```
## Проблематика  
Администраторы сталкиваются с необходимостью точного определения версии junction-компонентов IAM Proxy для диагностики, обновления или интеграции. Отсутствие централизованного источника информации может привести к ошибкам в управлении зависимостями между модулями.

## Предусловия  
1. Доступ к PACMAN (CFGA) с правами администратора.  
2. Знание расположения конфигурационных файлов junction.  
3. Учетные записи с правами на чтение API junction (при необходимости).  

## Правила эксплуатации  
1. **Файлы PACMAN**: Не изменяйте поля `version` и `build_date` вручную — они используются для автоматической проверки совместимости компонентов.  
2. **API**: Используйте только официальные endpoints для получения информации о версии.  
3. **Логи**: Избегайте извлечения версии из системных логов — это может нарушить их целостность.  

## Результат  
Администратор получает точную информацию о версии junction-компонента, включая:  
- основной номер версии (например, `2.4.1`);  
- хэш коммита сборки (для отладки);  
- дату сборки (для аудита).  

## Описание механизмов безопасности   
1. **Ограничение доступа**:  
   - файлы конфигурации PACMAN и API-endpoints доступны только для пользователей с правами администратора;  
   - все запросы к API шифруются через TLS.  

2. **Целостность данных**:  
   - хэш коммита (`commit_hash`) позволяет проверить оригинальность версии и исключить модификации.  

3. **Логирование**:  
   - все запросы к API-эндпоинту `/api/junction/version` фиксируются в логах IAM Proxy для аудита.
