# Вызов дополнительного отзыва токена. OIDC_REVOKE_REFRESH_TOKEN = True, OIDC_REVOKE_ACCESS_TOKEN = True

## Описание

При вызове логаута, IAM Proxy обращался к IDP 2 раза для каждого токена (по одному запросу на каждый токен)

## Предусловия

В env'ах настроены параметры:
OIDC_REVOKE_TOKENS_ON_LOGOUT = True
OIDC_REVOKE_REFRESH_TOKEN = True
OIDC_REVOKE_ACCESS_TOKEN = True
DEBUG = 1

В админке кейклока в разделе "события" на вкладке "Конфигурация" включить в "Сохраняемые типы событий" отображение REVOKE_GRANT и REVOKE_GRANT_ERROR

## Шаги проверки

1. **Действие**:

   Открыть админку кейклока на вкладке "События"

2. **Действие**:

   На соседней вкладке перейти на страницу прокси

   **Успешный результат**:

   Редирект на форму аутентификации

3. **Действие**:

   Пройти аутентификацию

   **Успешный результат**:

   Аутентификация выполнена успешно

4. **Действие**:

   Выполнить логаут на странице прокси

   **Успешный результат**:

   Окно подтверждения логаута отсутствует
   Логаут выполнен успешно

5. **Действие**:

   На вкладке "События" нажать кнопку "обновить"

   **Тестовые данные**:

   REVOKE_GRANT
   REVOKE_GRANT_ERROR

   **Успешный результат**:

   На вкладке отображаются два события:
   REVOKE_GRANT
   REVOKE_GRANT_ERROR

6. **Действие**:

   Перейти в логи IAMProxy и найти в логах записи о ревоуках refresh и access токенов

   **Тестовые данные**:

   "revocation of refresh_token successful"
   "revocation of access_token successful"

   **Успешный результат**:

   В логах есть запрос на ревоук
