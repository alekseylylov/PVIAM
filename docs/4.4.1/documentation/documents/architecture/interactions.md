| 3  | Proxy Server, IAM Proxy          | Балансировщик нагрузки IAM KCSE                | Балансировщик нагрузки направляет запрос на `Proxy Server` в момент его получения                                                                                                          |           HTTP           |                           |      TLS 1.2+       |
| 4  | Proxy Server, IAM Proxy          | Балансировщик нагрузки IAM AZGT                | Первое обращение к UI и API АС через `Proxy Server`, если это обращение требует авторизации в `Авторизация`                                                                                |           HTTP           |                           |      TLS 1.2+       |
| 5  | Proxy Server, IAM Proxy          | Подсистема с UI и API                          | Обращение к UI и API АС через `Proxy Server`                                                                                                                                               |           HTTP           |                           |      mTLS 1.2+      |
| 6  | Proxy Server, IAM Proxy          | RDS Server                                     | При периодическом получении/обновлении параметров проксирования для настройки `Proxy Server` (параметры в части ответвлений)                                                               |           HTTP           |         REST API          |      mTLS 1.2+      |
| 8  | Proxy Server, IAM Proxy          | Балансировщик нагрузки IAM AZGT                | Первое обращение к UI и API АС через `Proxy Server`, если это обращение требует авторизации в `Авторизация`                                                                                |           HTTP           |         REST API          |      mTLS 1.2+      |

| 8  | RDS Server                       | Audit (AUDT)                                   | В момент изменения активного контура на `RDS Server` выполняется отправка события в `Audit` (AUDT)                                                                                         |      jdbc или HTTP       |     REST API при HTTP     |      mTLS 1.2+      |
| 9  | RDS Server                       | PACMAN (CFGA)                                  | При старте `RDS Server` получает данные о параметрах для `Proxy Server` из `PACMAN` (CFGA), а после, периодически обновляет их                                                             |      Kafka или HTTP      |     REST API при HTTP     |      mTLS 1.2+      |
| 10 | RDS Server                       | Прикладной журнал (APLJ)                       | При старте `RDS Server` получает данные о активном контуре из `Прикладной журнал` (APLJ), а после, периодически обновляет их                                                               |      Kafka или HTTP      |     REST API при HTTP     |      mTLS 1.2+      |
| 11 | Proxy Server, IAM Proxy          | Audit (AUDT)                                   | По завершению обработки HTTP запроса к `Proxy Server` выполняется запись в журнал доступа, и по необходимости производится отправка события в `Audit` (AUDT)                               |           HTTP           |         REST API          |      mTLS 1.2+      |
| 12 | Балансировщик нагрузки IAM KCSE  | Провайдер аутентификации OIDC, IAM KCSE (KCSE) | Балансировщик нагрузки направляет запрос на `Провайдер аутентификации OIDC` в момент его получения                                                                                         |           HTTP           |         REST API          |      mTLS 1.2+      |
| 13 | Балансировщик нагрузки IAM Proxy | Proxy Server, IAM Proxy                        | Балансировщик нагрузки направляет запрос на `Proxy Server` в момент его получения                                                                                                          |           HTTP           |         REST API          |      mTLS 1.2+      |
| 14 | Балансировщик нагрузки IAM AZGT  | Авторизация (AZGT)                             | Балансировщик нагрузки направляет запрос на `Авторизация` в момент его получения                                                                                                           |           HTTP           |         REST API          |      mTLS 1.2+      |
| 15 | RDS Server                       | Журналирование (LOGA)                          | При появлении записей в журнале работы `RDS Server` они передаются в `Журналирование` (LOGA)                                                                                               |      Kafka или HTTP      |     REST API при HTTP     |      mTLS 1.2+      |
| 16 | Proxy Server, IAM Proxy          | Внешняя система управления секретами (Secman)  | При старте `Proxy Server` получает секреты из `Внешней системы управления секретами` (Secman), а после, периодически проверяет их изменение и обновляет по необходимости                   |           HTTP           |         REST API          |      mTLS 1.2+      |                         

Примечания:
- вызовы к `Proxy Server`, `Keycloak.SE` и `Авторизация (AZGT)` осуществляются
  через балансировщики нагрузки.
