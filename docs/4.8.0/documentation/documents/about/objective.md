# Назначение

*Platform V IAM SE* используется во время взаимодействия пользователей с бизнес-приложениями (runtime). Для обеспечения
выполнения функций по администрированию учетных записей, управлению их полным жизненным циклом (admin time) необходимо
использовать дополнительные к *Platform V IAM SE* инструменты Identity Management/Identity Governance.

*Platform V IAM SE* содержит набор инструментов для управления доступом к информационным ресурсам. Данные
инструменты необходимы для аутентификации и авторизации пользователей. Они обеспечивают удобство подключения защищаемых
приложений к провайдеру идентификации, поддерживают реализацию множества различных сценариев работы в соответствии с
отраслевыми стандартами, такими как OpenID Connect, Oauth 2.0, SCIM 2.0.

## Состав продукта

В состав **Platform V IAM SE (IAM)** входят следующие компоненты:

### IAM Proxy (AUTH)

IAM Proxy - предназначен для проксирования веб-запросов, идентификации и аутентификации пользователей при доступе к
приложениям, в том числе:

1. Создание аутентификационной сессии.
2. Управление аутентификационной сессией.
3. Взаимная аутентификация (mTLS) с защищаемым бэкенд.
4. Терминирование TLS.
5. Реверсивное проксирование.
6. Обогащение запроса сведениями о пользователе.
7. Обработка запросов на аутентификацию от пользователей, а при необходимости - использование принудительного запроса
  аутентификационных данных.
8. Передача идентификатора пользователя для нужд идентификации в подсистемах.
9. Формирование событий аудита в подсистемы аудита для их последующей обработки.
10. Ограничение доступа не аутентифицированного пользователя к подсистемам.
