# Чек-лист проверки корректности работы

После окончания развертывания IAM Proxy, необходимо проверить корректность развертывания, как самого IAM Proxy, так и
его компонентов.

*В IAM Proxy реализованна проверка компонентов на этапе развертывания, в случае ошибки, развертывание останавливается и
возвращает пользователю сообщение об ошибке.*

## Проверка балансировщика

Для проверки работы балансировщика, необходимо выполнить следующие действия:

1. Перейти на точку входа (балансировщик IAM Proxy) заданную на этапе развертывания IAM
   Proxy https://platform-ift2.mycompany.ru/proxy/ (подробнее в
   разделе [Установка](../installation-guide/installation.md).
   - если на IAM Proxy настроено проксирование по корню, то перейдите на https://platform-ift2.mycompany.ru/proxy/.
2. Откроется форма аутентификации пользователя.
3. Пройти аутентификацию, введя имя учетной записи и пароль.
4. Убедитесь, что после успешной аутентификации происходит перенаправление на стартовую страницу IAM Proxy — это
означает корректную работу провайдера (Keycloak) и самого IAM Proxy.

Если используется балансировщик из поставки Platform V IAM, то доступность сервиса балансировщика можно проверить,
запросив на сервере с установленным сервисом (systemd unit `lb-iamproxy`) его статус:

`systemctl status lb-iamproxy`

Пример ответа:

```
lb-iamproxy.service - L4 Load balancer on Nginx.
Active: active (running) since Thu 2023-04-13 16:58:54 MSK; 1 months 9 days ago`
...
```

Проверим работоспособность сервиса, запросив список опубликованных портов на виртуальной машине балансировщика:

`netstat -ntlp | grep -i lb-iamproxy`

```
  tcp        0      0 0.0.0.0:443                 0.0.0.0:*                   LISTEN      970981/lb-iamproxy.con
  tcp        0      0 127.0.0.1:10082             0.0.0.0:*                   LISTEN      970981/lb-iamproxy.con
  tcp        0      0 0.0.0.0:80                  0.0.0.0:*                   LISTEN
```

## Проверка IAM Proxy

Для проверки работы IAM Proxy (это systemd unit `iamproxy`), необходимо выполнить следующие действия:

В случае использования архитектурной схемы без балансировщика можно провести валидацию без него, для этого требуется
выполнить следующие шаги:

1. Перейти на точку входа (сервер IAM Proxy) заданную на этапе развертывания IAM
   Proxy https://platform-ift2.mycompany.ru/proxy/ (подробнее в
   разделе [Установка](../installation-guide/installation.md).
   - если на IAM Proxy настроено проксирование по корню, то перейдите на https://platform-ift2.mycompany.ru/proxy/.
2. Откроется форма аутентификации пользователя.
3. Пройти аутентификацию, введя имя учетной записи и пароль.
4. Убедитесь, что после успешной аутентификации происходит перенаправление на стартовую страницу IAM Proxy — это
означает корректную работу провайдера (Keycloak) и самого IAM Proxy.

Доступность сервиса можно проверить, запросив статус IAM Proxy (systemd unit `IAM Proxy`) командой:

`systemctl status iamproxy`

Пример ответа:

```
iamproxy.service - Platform V IAM Proxy. Reverse proxy with authentifacation by Open ID Connect.
Active: active (running) since Tue 2023-05-23 10:51:21 MSK; 1h 3min ago
```

Для проверки работоспособности сервиса, запросим список открытых портов на виртуальной машине IAM Proxy:

`netstat -ntlp | grep -i syngx`

```
  tcp        0      0 0.0.0.0:10443               0.0.0.0:*                   LISTEN      2811987/syngx
  tcp        0      0 0.0.0.0:10444               0.0.0.0:*                   LISTEN      2811987/syngx
  tcp        0      0 0.0.0.0:10080               0.0.0.0:*                   LISTEN      2811987/syngx
```

## Проверка RDS Client

Для проверки работы RDS Client, нужно запросить ssh-доступ к хостам IAM Proxy (так как RDS Client используется только
хостами IAM Proxy). Доступность сервиса можно проверить, запросив статус systemd unit RDS Client (`rds-client`):

`systemctl status rds-client`.

Ожидаемый ответ вывода команды:

```
rds-client.service - RDS client for nginx (for IAM Proxy)
Active: active (running) since Thu 2023-05-18 13:34:55 MSK; 4 days ago
```

## Проверка Fluent Bit

Функциональность Fluent Bit, можно проверить используя административную панель Platform V Audit SE, путем поиска событий
отправляемых компонентами стенда в ТС Аудит:

`systemctl status fluent-bit`

Пример ответа:

```
● fluent-bit.service - Fluent Bit (for Platform V IAM Proxy)

     Active: active (running) since Fri 2025-11-21 22:19:37 MSK; 42min ago

      Tasks: 11 (limit: 24756)
     Memory: 42.9M
        CPU: 1.501s
     CGroup: /system.slice/fluent-bit.service
             └─665693 /usr/bin/fluent-bit -c /etc/fluent-bit/fluent-bit.conf
```

## Проверка работоспособности интеграций

### Проверка интеграции с Platform V Audit SE (AUDT)

Для проверки интеграции с Platform V Audit SE выполните следующие действия:

1. Пройти аутентификацию в IAM Proxy.
2. Перейти на стартовую страницу IAM Proxy и выбрать свою АС.
3. Перейти в UI Platform V Audit SE.
4. Отфильтровать список событий по желаемому параметру (например: контекст по которому запрашивается АС) и задать
   временной интервал и выполнить поиск.
5. Убедитесь в успешной выдаче результатов.

### Проверка интеграции с Журналирование LOGA

Для проверки интеграции с Журналирование LOGA выполните следующие действия:

1. Перейти в UI Журналирование LOGA.
2. Отфильтровать список событий по желаемому параметру (например: контекст по которому запрашивается АС) и задать
   временной интервал и выполнить поиск.
3. Убедитесь в успешной выдаче результатов.

### Проверка интеграции с Keycloak.SE

Для проверки интеграции с Keycloak.SE выполните пункты 1 и 2 из подраздела Проверка IAM Proxy.

### Проверка интеграции с внешней системой хранения секретов (SecMan)

Для проверки интеграции с внешней системой хранения секретов (SecMan) на VM с установленным программным компонентом IAM
Proxy, необходимо выполнить следующие действия:

1. Запросить в терминале статус `systemd unit` vault-agent посредством команды `systemctl status -l vault-agent`.
   Ожидаемый ответ "Active: active (running)".
2. Убедитесь в наличии секретов в целевой директории хранения vault-agent.

Для проверки интеграции с внешней системой хранения секретов (SecMan) в Openshift или DropApp с установленным
программным компонентом IAM Proxy, необходимо выполнить следующие действия:

1. Убедитесь в доступности контейнера `vault-agent` в pod'е IAM Proxy через консоль OpenShift или DropApp.
2. Убедитесь в наличии секретов в целевой директории хранения vault-agent контейнера `iamproxy` pod IAM Proxy.

### Проверка интеграции с Авторизацией (AZGT)

Для проверки работоспособности Авторизации, выполните следующие шаги:

1. Выполнить В терминале get-запрос к API Авторизации. Пример запроса:
   ```shell
   curl -k https://ufs-security-pub2-tribe-sc-autz-ift-2.apps.ocp.devpub.solution.mycompany.ru/healthcheck/service-healthcheck/ufs-security \
     --cert клиентский_сертификат.crt --key клиентский_ключ.key
   ```

2. Дождитесь ответа (ожидаемый код ответа 200).
   Пример ответа:
   ```json
   {"success":true,"body":"ON"}
   ```
