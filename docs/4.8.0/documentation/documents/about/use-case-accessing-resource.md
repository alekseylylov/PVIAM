# Получить ресурс требующий аутентификации

## Основной сценарий

1. Аутентифицированный пользователь обращается через IAM Proxy к ресурсу требующему аутентификации.
2. Аутентифицированный пользователь получает доступ к ресурсу.

## Альтернативный сценарий

1. Аутентифицированный пользователь обращается через IAM Proxy к ресурсу требующему аутентификации.
2. Сессия или токены не действительны.
3. IAM Proxy перенаправляет пользователя в провайдер аутентификации, чтобы пройти аутентификацию (
   см. [Пройти аутентификацию](use-case-authentification.md)).
4. После успешного прохождения аутентификации пользователь получает доступ к ресурсу.

## Альтернативный сценарий

1. Не аутентифицированный пользователь обращается к IAM Proxy указав JWT-токен необходимый для аутентификации в HTTP
   заголовке `Authorization`, чтобы использовать аутентификацию по JWT-токену.
2. IAM Proxy проверяет токен, и получает данные с запрошенного ресурса.
3. Пользователь получает запрошенный ресурс от IAM Proxy.

## Альтернативный сценарий

1. Не аутентифицированный пользователь обращается к IAM Proxy указав данные необходимые для аутентификации в HTTP
   заголовке `Authorization`, чтобы использовать аутентификацию direct-auth.
2. Сессия или токены не действительны.
3. IAM Proxy перенаправляет пользователя в провайдер аутентификации, чтобы пройти аутентификацию (
   см. [Пройти аутентификацию](use-case-authentification.md)).
4. После успешного прохождения аутентификации пользователь получает доступ к ресурсу.

## Исключительный сценарий

1. Аутентифицированный пользователь обращается через IAM Proxy к ресурсу требующему аутентификации.
2. Сессия или токены не действительны.
3. IAM Proxy возвращает ответ с HTTP кодом `401`(код может отличаться в зависимости от причины отказа).

## Исключительный сценарий

1. Пользователь обращается к ресурсу через IAM Proxy.
2. Ресурс временно недоступен (например, из-за технического обслуживания или высокой нагрузки).
3. IAM Proxy возвращает ответ с HTTP кодом `503`(код может отличаться в зависимости от причины недоступности).
