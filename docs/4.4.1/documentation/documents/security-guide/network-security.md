# Сетевая безопасность

Сетевая безопасность обеспечивается Platform V Synapse Service Mesh (Ingress/Egress). Сетевая безопасность для
виртуальных машин обеспечивается компонентом IAM Proxy.

Рекомендуется во всех вариантах развертывания:

- использовать HTTPs при подключении из браузера до IAM Proxy (допускается не использовать клиентский сертификат при подключении, в k8s режим `SIMPLE` в `kind: Ingress`);
- не терминировать TLS на промежуточных узлах от браузера до IAM Proxy;
- для передачи данных от браузера до IAM Proxy использовать протокол TLS версии 1.2 и выше.

Строго рекомендуется во всех вариантах развертывания:

- использовать аутентификацию по клиентскому сертификату для доступа пользователя в консоль администратора.

## Рекомендации по защите сети и передаваемых данных
Для обеспечения безопасности передаваемых данных используется **криптографический протокол взаимной (двусторонней) аутентификации mTLS 1.2**. Передача данных между компонентами IAM Proxy (в том числе через контейнеризированную среду) осуществляется с обязательной проверкой сертификатов клиентов и серверов.

Cookie, связанные с сессиями пользователей, передаются с флагами **HttpOnly** и **Secure**, исключающими возможность доступа к значениям cookie из сценариев и ограничивающими их передачу только по безопасному протоколу HTTPS.

Для выходных данных используется **Content-Type application/json**, что предотвращает выполнение скриптов при непосредственном отображении JSON в браузере. Экранирование специальных символов при отображении в интерфейсе администрирования выполняется с помощью стандартных средств React, что минимизирует риски XSS-атак.

## Настройки защиты сети
Для настройки входящих (ingress) и исходящих (egress) соединений воспользуйтесь инструкциями из документа [Документация по SSM](https://client.sbertech.ru/docs/public/SSM/5.8.0/index.html):
- «Настройка интеграции с компонентом Сервисный прокси продукта Platform V Synapse Service Mesh»;
- «Настройка интеграции с компонентом Граничный прокси продукта Platform V Synapse Service Mesh».  
  
Для настройки **mTLS** воспользуйтесь инструкцией из документа [Дополнительные материалы](../additional-materials/rds-server-integration-mtls-conf.md):
- «Настройка подключения по mTLS к базе данных PostgreSQL».
- «Конфигурация доверенных сертификатов для межсервисной авторизации».

> **Примечание**: Все секреты (например, `proxy_oidc_client_secret`) хранятся в **SecMan** (аналог HashiCorp Vault), что исключает их хранение в конфигурационных файлах в открытом виде.
