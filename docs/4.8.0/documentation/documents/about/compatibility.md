# Совместимость с выпущенными клиентами

## Клиент 1: SberID Mobile App
### PURL  
`pkg:generic/sberid-mobile@4.3`

### Поддерживаемые протоколы  
- **OpenID Connect** (OIDC);  
  **OAuth 2.0**.

### Версия IAM Proxy  
- совместима с версиями `4.5.0+`

### Особенности интеграции  
- поддержка offline-аутентификации по одноразовым кодам;  
- интеграция с HSM для хранения приватных ключей;  
- использование Push-уведомлений для подтверждения входа;  
- автоматическое обновление токенов в фоне.

## Клиент 2: Google Authenticator
### PURL  
`pkg:generic/google-authenticator@latest`

### Поддерживаемые протоколы  
- **TOTP** (Time-based One-Time Password)

### Версия IAM Proxy  
- совместима с версиями `4.3.0+`

### Особенности интеграции  
- не требует дополнительной настройки на стороне IAM Proxy;  
- используется как резервный метод MFA;  
- поддерживается импорт ключей через QR-код;  
- рекомендуется к использованию в связке с основным методом аутентификации.

## Клиент 3: Keycloak Admin CLI
### PURL  
`pkg:generic/keycloak-cli@23.0.6`

### Поддерживаемые протоколы  
- **REST API**;  
- **OAuth 2.0 Client Credentials**.

### Версия IAM Proxy  
- совместима с версиями `4.6.0+`

### Особенности интеграции  
- полная поддержка управления пользователями, ролями и клиентами через CLI;  
- интеграция с Platform V IAM SE через защищенные endpoint;  
- поддержка выполнения скриптов автоматизации;  
- логирование всех административных операций.

## Клиент 4: Kubernetes RBAC Plugin
### PURL  
`pkg:generic/k8s-rbac@1.28`

### Поддерживаемые протоколы  
- **Kubernetes TokenReview API**;  
- **OIDC Discovery**.

### Версия IAM Proxy  
- совместима с версиями `4.7.0+`

### Особенности интеграции  
- динамическая авторизация на основе ролей IAM;  
- поддержка OpenShift 4.12+ и Kubernetes 1.28+;  
- кеширование токенов для снижения нагрузки на IAM Proxy;  
- интеграция с audit-логами кластера.

## Клиент 5: Веб-приложение (Single Page Application)
### PURL  
`pkg:npm/@platformv/web-client@1.5.1`

### Поддерживаемые протоколы  
- **OpenID Connect** (OIDC)  
- **OAuth 2.0**

### Версия IAM Proxy  
- совместима с версиями `4.6.0+`

### Особенности интеграции  
- автоматическая обработка токенов JWT;  
- поддержка SSO через ЕСИА, VK, Mail.ru, SberID;  
- интеграция с ABAC/PBAC-политиками для ограничения доступа к ресурсам;  
- динамическая загрузка политик через RDS API без перезагрузки приложения.

## Клиент 6: Десктопное приложение (Electron)
### PURL  
`pkg:electron/platformv-desktop@1.0.0`

### Поддерживаемые протоколы  
- **OpenID Connect** (OIDC)  
- **OAuth 2.0**

### Версия IAM Proxy  
- совместима с версиями `4.3.5+`

## Клиент 7: API-сервис (REST/GraphQL)
### PURL  
`pkg:docker/platformv/api-gateway@3.1.2`

### Поддерживаемые протоколы  
- **OpenID Connect** (OIDC)  
- **JWT** (JSON Web Token)

### Версия IAM Proxy  
- совместима с версиями `4.7.0+`

### Особенности интеграции  
- гранулированная авторизация через ABAC-политики;  
- логирование и аудит всех запросов с отправкой в Platform V Audit SE;  
- интеграция с Prometheus и Fluent Bit для мониторинга метрик и логов;  
- поддержка rate-limiting на уровне IAM Proxy для защиты от DDoS.

## Клиент 8: Система управления контентом (CMS)
### PURL  
`pkg:deb/ubuntu/platformv-cms@1.3.0`

### Поддерживаемые протоколы  
- **SAML 2.0**  
- **LDAP** (через адаптер IAM Proxy)

### Версия IAM Proxy  
- совместима с версиями `4.4.0+`

### Особенности интеграции  
- централизованное управление правами доступа;  
- поддержка ролей и групп для редактирования контента;  
- интеграция с PKI CA для шифрования данных;  
- добавлена поддержка аудита изменений через журнал событий IAM.
