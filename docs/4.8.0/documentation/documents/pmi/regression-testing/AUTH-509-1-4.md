# SecMan. OSE. IAM Proxy. Все секреты вынесены в отдельный файл oidc-auth-secrets.server.conf

## Описание

## параметры приемки

1. Проверить, что Deployment сервиса содержит аннотации подключения Vault-agent сайдкара.
2. Провести замену секретов в Secman и дождаться обновления секретов на подах.
3. Удостовериться, что секреты сервиса IAM Proxy изменились на новые.

## Предусловия

1. Проверить, что Deployment сервиса содержит аннотации подключения Vault-agent сайдкара.

## Шаги проверки

1. **Действие**:

   В Deployments основного модуля auth-iamproxy секреты заданы в виде аннотаций
   {{- with secret "..." -}}

   **Тестовые данные**:

   vault.hashicorp.com/agent-inject-template-oidc-auth-secrets: |
    {{- with secret "DEV_DZO/A/DEV/AUTH/KV/OSE.pub2-tribe-sc-auth-ift-1" -}}
    {{- $proxy_session_secret := or .Data.PROXY_SESSION_SECRET "" -}}
    {{- $proxy_oidc_client_id := or .Data.PROXY_OIDC_CLIENT_ID "" -}}
    {{- $proxy_oidc_client_secret := or .Data.PROXY_OIDC_CLIENT_SECRET "" -}}
    {{- $oidc_client_rsa_private_key := or .Data.OIDC_CLIENT_RSA_PRIVATE_KEY "" -}}
    {{- $authz_by_oauth_jwt_public_key := or.Data.AUTHZ_BY_OAUTH_JWT_PUBLIC_KEY "" -}}
    {{- $authz_spas_secret := or .Data.AUTHZ_SPAS_SECRET "123456" -}}

2. **Действие**:

   В консоли OSE перейти на вкладку Workloads -> Pods. Выбрать pod и перейти
   на вкладку Terminal.

   **Успешный результат**:

   Выполнено

3. **Действие**:

   В Terminal выполнить команду просмотра содержимого секретов по пути
   /secrets/oidc-auth-secrets.server.conf

   **Тестовые данные**:

   Файл можно посмотреть с помощью команды cat.
   Синтаксис команды: cat {путь к файлу}

   **Успешный результат**:

   В терминале выведено содержимое файла.
