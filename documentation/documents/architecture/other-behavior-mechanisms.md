# Прочие поведенческие механизмы

> Вариант администрирования с помощью PACMAN (CFGA) является устаревшим (deprecated).
> Для администрирования с использованием PACMAN (CFGA) или yaml-файла в формате PACMAN, требуется наличие RDS Server на
> стенде. Поддержку данной интеграции и RDS Server планируется прекратить с релиза Platform V IAM 5.0.0.

Для осуществления проксирования на конкретное приложение необходимо на новом или уже существующем сервере IAM Proxy
добавить ответвление. Ответвления добавляются на стороне компонента PACMAN (CFGA) администраторами сервисов платформы.
После применения изменений в компоненте PACMAN (CFGA), в течение 5 секунд изменения должны примениться. RDS Server
отслеживает конфигурацию в своем артефакте (c ID `rds-server`) из компонента PACMAN (CFGA) и передает сервису IAM Proxy
конфигурацию. Изменения применяются без необходимости перезагрузки.

> Примечание
> Для изменения периода вычитки конфигурации, измените значение для параметра `checkConfigurationFrequency` в PACMAN
> (CFGA) или yaml-файле, на необходимое.

IAM Proxy предоставляет возможность использовать в своих веб-приложениях следующие возможности:

- `WebSocket` и `Server-send events`;
- взаимодействие с внешним провайдером аутентификации по `OpenID Connect`;
- авторизация доступа к защищаемым приложениям по ролям: дает возможность ограничить обращение к бэкенд приложениям на
  основе ролей из токена (по маске роли, регулярное выражение);
- балансировка (hash по `username`+`real ip`);
- хранение HTML-статики.

На IAM Proxy кешируются метаданные провайдера аутентификации `OpenID Connect` (время жизни кеша 24 часа).
Метаданные включают (данные получаемые от провайдера) описание возможностей и точек входа в провайдер аутентификации.

Пользовательские данные на IAM Proxy не кешируются.
