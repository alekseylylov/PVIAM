# Порядок расчета сайзинга

Объем КТС для каждого компонента необходимо рассчитывать, исходя из анализа диаграмм развертывания и объемов КТС,
полученных с помощью калькулятора (Platform V Cost Calculator) с параметрами сайзинг-модели.

1. Определить перечень компонентов, используемых в решении Platform V IAM, исходя из анализа диаграмм развертывания
   и/или архитектуры решения.
2. Произвести для каждого используемого компонента расчеты объемов КТС с помощью калькулятора с параметрами
   сайзинг-модели [Расчет сайзинга компонентов](sizing-md-components.md).

По результатам расчета необходимых КТС для каждого компонента определяется сайзинг следующих ресурсов:

| Код компонента | Имя компонента                  | POD CPU | POD RAM | VM CPU | VM RAM | VM HDD |
|----------------|---------------------------------|:-------:|:-------:|:------:|:------:|:------:|
| AUTH           | Прокси сервер                   |    X    |    X    |   X    |   X    |   X    |
| KCSE           | Аутентификация                  |    X    |    X    |   X    |   X    |   X    |

Типы ресурсов используемых для компонентов:

| Код компонента | Имя компонента | Контейнеры | БД  | Kafka |
|----------------|----------------|:----------:|:---:|:-----:|
| AUTH           | Прокси сервер  |     X      |  X  |   X   |
| KCSE           | Аутентификация |     X      |  X  |   X   |

3. Произвести суммирование всех полученных сайзингов в разрезе каждого ресурса (POD CPU, POD RAM, VM CPU, VM RAM,
   VM HDD).

Расчет дополнительных КТС, необходимых для sidecar: Сервисный прокси (SVPX), Сервис сбора метрик MONA, istio-proxy
(ingress/egress), ott – осуществляется в калькуляторе соответствующего компонента (сайзинг перечисленных sidecar
зависит от нагрузки на соответствующий компонент).
