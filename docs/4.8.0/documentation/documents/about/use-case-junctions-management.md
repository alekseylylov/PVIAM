# Управление ответвлениями

## Основной сценарий

1. Администратор проходит аутентификацию в UI управления ответвлениями (RDS API или UI приложения).
2. Администратор в UI выполняет необходимые действия с ответвлениями (создает/изменяет/удаляет).
3. Измененные данные по ответвлениям применяются при периодическом обновлении с IAM Proxy.

## Альтернативный сценарий

1. Администратор управляет ответвлениями через текстовый yaml-файл на RDS API.
2. Администратор сохраняет изменения.
3. Измененные данные по ответвлениям применяются при периодическом обновлении с IAM Proxy.

## Исключительный сценарий

1. Администратор проходит аутентификацию в UI управления ответвлениями RDS API или UI приложения).
2. Администратор вводит некорректные данные необходимые для аутентификации.
3. Провайдер аутентификации сообщает об ошибке. Не удается аутентифицировать пользователя.

## Исключительный сценарий

1. Администратор проходит аутентификацию в UI управления ответвлениями (RDS API или UI приложения).
2. Администратор в UI выполняет необходимые действия (создает/изменяет/удаляет) с ответвлениями.
3. Администратор задает некорректные данные по ответвлениям.
4. При периодическом обновлении с IAM Proxy не проходит тест конфигурации, и измененные данные по ответвлениям не
   применяются (IAM Proxy продолжает работу на предыдущей рабочей конфигурации).
