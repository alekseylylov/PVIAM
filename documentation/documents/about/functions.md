# Основные функции

| Название функции                                                                                                                                               | Потребитель функции                       | Аргументы функции                                   | Результат                                                                                                                                                                                                                             |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------|-----------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Аудит                                                                                                                                                          | Сотрудник кибербезопасности/аудита        | Информация об объекте аудита                        | События аутентификации, как успешные, так и неуспешные. Изменения настроек IAM                                                                                                                                                        |
| Журналирование                                                                                                                                                 | Сотрудник сопровождения/кибербезопасности | Журналы состояний системы                           | Журналы конфигурирования и трассировки запросов, показатели текущего состояния сервиса                                                                                                                                                |
| Обработка запросов требующих наличия аутентификации                                                                                                            | Пользователь                              | Идентификатор (URL) ресурса                         | Аутентификация пользователей по паролю. Аутентификация пользователей по Open ID Connect. Аутентификация пользователей на прокси-сервере IAM                                                                                           |
| Передача идентификатора пользователя в подсистемах                                                                                                             | Подсистемы                                | Информация о пользователе в формате JWT             | Идентификация в подсистемах                                                                                                                                                                                                           |
| Ограничение доступа не аутентифицированного пользователя к подсистемам                                                                                         | Владелец подсистемы                       | Аутентификационные данные                           | Ограничение доступа пользователю                                                                                                                                                                                                      |
| Обработка запросов требующих аутентификации                                                                                                                    | Пользователь                              | Идентификатор(URL) ресурса                          | Попытка аутентификации входящего запроса. В случае успеха передача запроса следующему фильтру c данными аутентифицированного пользователя. В случае ошибки ответ с HTTP-статус кодом 403 (настраиваемый параметр)                     |
| Обработка запросов не требующих обязательной аутентификации                                                                                                    | Пользователь                              | Идентификатор(URL) ресурса                          | Попытка аутентификации входящего запроса. В случае успеха передача запроса следующему фильтру c данными аутентифицированного пользователя. В случае ошибки - передача запроса следующему фильтру с данными пользователя по умолчанию  |
| Валидация данных пользователя и проверка его прав, по источникам данных (JWT-token, сессионная cookie, токен аутентификации) передаваемых в заголовке запроса  | Пользователь                              | JWT-token, сессионная cookie, токен аутентификации  | Успешная/неуспешная валидация данных пользователя передаваемых в заголовке, сопоставление данных с актуальными правами                                                                                                                |
