# Совместимость с выпущенными клиентами

## Клиент 1: Web-приложение (Single Page Application)
### PURL  
`pkg:npm/@platformv/web-client@1.4.0`

### Поддерживаемые протоколы  
- **OpenID Connect** (OIDC); 
- **OAuth 2.0**.

### Версия IAM Proxy  
- совместима с версиями `4.6.0+`.  

### Особенности интеграции  
- автоматическая обработка токенов JWT;  
- поддержка SSO (Single Sign-On) через ЕСИА, VK, Mail.ru;  
- интеграция с ABAC/РBAC-политиками для ограничения доступа к ресурсам.  

## Клиент 2: Мобильное приложение (Android/iOS)
### PURL  
`pkg:android/com.platformv.mobile@2.1.3`

### Поддерживаемые протоколы  
- **OpenID Connect** (OIDC); 
- **SAML** (через обратный прокси).  

### Версия IAM Proxy  
- совместима с версиями `4.5.0+`. 

### Особенности интеграции  
- многофакторная аутентификация (MFA) через TOTP;  
- кэширование токенов для офлайн-режима;  
- поддержка Push-уведомлений для подтверждения действий.  

## Клиент 3: API-сервис (REST/GraphQL)
### PURL  
`pkg:docker/platformv/api-gateway@3.0.0`

### Поддерживаемые протоколы  
- **OpenID Connect** (OIDC)  
- **JWT** (JSON Web Token)  

### Версия IAM Proxy  
- Совместима с версиями `4.7.0+`  

### Особенности интеграции  
- гранулированная авторизация через ABAC-политики;  
- логирование и аудит всех запросов;  
- интеграция с Prometheus для мониторинга метрик.  

## Клиент 4: Система управления контентом (CMS)
### PURL  
`pkg:deb/ubuntu/platformv-cms@1.2.0`

### Поддерживаемые протоколы  
- **SAML**; 
- **LDAP** (через адаптер IAM Proxy). 

### Версия IAM Proxy  
- совместима с версиями `4.4.0+`.  

### Особенности интеграции  
- централизованное управление правами доступа;  
- поддержка ролей и групп для редактирования контента;  
- интеграция с PKI CA для шифрования данных.  

## Клиент 5: Десктопное приложение (Electron)
### PURL  
`pkg:electron/platformv-desktop@0.9.8`

### Поддерживаемые протоколы  
- **OpenID Connect** (OIDC);  
- **OAuth 2.0**.  

### Версия IAM Proxy  
- совместима с версиями `4.3.5+`.  

### Особенности интеграции  
- безопасное хранение токенов в локальном хранилище; 
- автоматическое обновление токенов при истечении срока действия;  
- поддержка темизации интерфейса (светлая/темная).
