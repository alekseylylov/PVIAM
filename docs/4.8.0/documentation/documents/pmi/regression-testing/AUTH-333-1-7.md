# Метрики Prometheus. Мониторинг задержек запросов\ответов от backend

## Предусловия

1) Метрики должны быть включены в параметрах деплоя. В файле
install_eip/config/ST_ALT_GT/group_vars/proxy/proxy.yml установить значение
параметра proxy_metrics_enable: True (True - собирать метрики и
опубликовать их в формате Prometeus на<http://127.0.0.1:10080/metrics)> и
выполнить передеплой на стенд
2) Настроить в Конфигураторе junction с конфигурацией. Для этого требуется
Запустить АС Конфигуратор (ввести логин и пароль администратора) ->
Осуществить поиск параметров конфигурации RDS (Управление маршрутами
прокси-сервера (Route Discovery Service)->{версия}->Root)-> Раскрыть группу
Параметры проксирования на back-end (JunctionConfig) -> В нее добавить
новую группу с конфигурацией:

{
 "junctionName": "Prometheus",
 "https": false,
 "indexUrl": "",
 "sslCommonName": ".sbrf.ru",
 "transparent": false,
 "applyJctRequestFilter": "",
 "authorizeByRoleTemplate": "",
 "junctionPoint": "/metrics",
 "serverAddresses": [
 "127.0.0.1:10080:11091261-5b03-0000-0000-31243614011e"
 ]
 },

## Шаги проверки

1. **Действие**:

   Открыть тестовое приложение proxy. Ввести логин и пароль пользователя

   **Успешный результат**:

   Аутентификация пройдена успешно

2. **Действие**:

   Перейти в приложение по ссылке Prometheus

   * После обновления конфигуратора задержка может составлять 30 сек

   **Успешный результат**:

   Отображена страница с метриками по адресу
   https://platform-st-alt-gt.solution.gt/metrics

3. **Действие**:

   Найти раздел с метриками мониторинга - гистограмма по задержкам

   # HELP http_durations Backend, гистограмма времени обработки HTTP запросов разрезе upstream+upstream_server (в секундах)
   # TYPE http_durations histogram

   **Успешный результат**:

   Метрика найдена

4. **Действие**:

   В записи метрики указаны http_durations_bucket:
    * upstream
    * upstream_addr
    * le
    * количество запросов
   Пример записи приложен в TestData

   **Тестовые данные**:

   http_durations_bucket{upstream="",upstream_addr="",le="+Inf"} 710
   http_durations_bucket{upstream="backend_jct_admin-cm",upstream_addr="172.27.227.86:9443",le="0.01"} 1

   **Успешный результат**:

   В метрики указаны данные о
    * upstream
    * upstream_addr
    * le
    * количество запросов для всех строк указано
    * все параметры инициализированы
