# Fallback_Authentication_Pattern

## Контекст
IAM Proxy может зависеть от внешних систем аутентификации, таких как Keycloak, Auth0 или Vault. При этом эти системы могут временно недоступны.

## Проблема
При отказе внешней системы аутентификации IAM Proxy становится непроизводительным, что может повлечь за собой остановку всего приложения.

## Решение
Реализовать паттерн "Fallback Authentication", при котором IAM Proxy использует внутреннюю стратегию аутентификации (например, простую проверку API ключа или хранение пользователей в локальной базе) в случае недоступности внешней системы. Такой подход обеспечивает отказоустойчивость и возможность ограниченного функционирования при частичном отказе инфраструктуры.
